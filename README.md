# NotebookLM Local Implementation

A Python-based local implementation of a NotebookLM-like workflow. This project allows you to process text files, generate summaries, convert them into structured tutorials, and synthesize them into audio format using Text-to-Speech (TTS).

## 🚀 Features

- **Text Processing**: Reads and processes local text files.
- **AI Summarization**: Generates concise summaries of long-form content (Placeholder for LLM integration).
- **Tutorial Conversion**: Transforms summaries into structured, educational tutorial formats.
- **Local Text-to-Audio**: Converts the generated tutorial into an MP3 audio file using **Coqui TTS**.
- **Privacy Focused**: Designed for local execution.

## 🛠️ Requirements

- Python 3.8+
- [Coqui TTS](https://github.com/coqui-ai/TTS)
- Dependencies listed in `requirements.txt`

## 📦 Installation

1. **Clone the repository**:
   ```bash
   git clone https://github.com/yourusername/NotebookLM.git
   cd NotebookLM
   ```

2. **Install dependencies**:
   ```bash
   pip install -r requirements.txt
   ```

   *Note: Coqui TTS may require additional system dependencies (like `espeak-ng`). Refer to the [Coqui TTS documentation](https://tts.docs.coqui.ai/en/latest/installation.html) for detailed setup.*

## 📖 Usage

1. Run the main script:
   ```bash
   python notebook_lm.py
   ```

2. Follow the prompts:
   - Provide the path to a `.txt` file (e.g., `sample_text.txt`).
   - The script will display the generated summary and tutorial.
   - An audio file (`tutorial_audio.mp3`) will be generated in the project directory.

## 📂 Project Structure

- `notebook_lm.py`: Main application logic.
- `requirements.txt`: Python dependencies.
- `sample_text.txt`: Example input file.
- `tutorial_audio.mp3`: Output audio file generated by the script.

## ⚙️ How it Works

The project follows a 4-step pipeline:
1. **Read**: Extracts text from a source file.
2. **Summarize**: Uses a placeholder logic to extract key information.
3. **Tutorialize**: Structures the summary into an easy-to-follow lesson plan.
4. **Synthesize**: Uses the `TTS` library to create a high-quality voiceover of the tutorial.
